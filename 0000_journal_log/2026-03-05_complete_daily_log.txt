  a) youtube_url was empty in database
  b) get_youtube_video_id() was returning None
  c) youtube_url field wasn't accepting the URL

Phase 3: Root Cause Analysis
- Checked forms.py: PostForm includes youtube_url field
- Checked create_post.html: Has "YouTube URL (Optional)" input field
- Checked models.py: youtube_url = models.URLField(blank=True, null=True)
- FOUND THE PROBLEM: URLField validates URLs strictly
- URLField rejects youtu.be URLs because it uses Django's built-in URL validator
- youtu.be is technically a valid URL but URLField may have issues with it
- Solution: Change URLField to CharField to accept any text

WHY CHANGES WERE NECESSARY
--------------------------

Problem 1: URLField Validation
Django's URLField uses a strict URL validator that works well for standard URLs
like https://www.youtube.com/watch?v=xxx but may reject or have issues with:
- youtu.be short URLs (different domain)
- URLs with special tracking parameters (?si=, ?t=)
- Non-standard URL formats

Solution: Use CharField instead, which accepts any string up to 500 characters
The extraction logic in get_youtube_video_id() method handles all URL formats anyway

Problem 2: Form Widget Validation
Django forms have two layers of validation:
1. Model field validation
2. Form widget validation

Even if model accepted the value, the form widget (URLInput) would reject it
before it ever reached the model.

Solution: Use TextInput widget which doesn't validate URL format
Let the model and Python code handle the validation instead

TECHNICAL NOTES
---------------

URLField vs CharField:
- URLField: Validates URL format, good for standard URLs
- CharField: No format validation, flexible for any string
- For YouTube URLs: CharField better because URLs are non-standard

TextInput vs URLInput:
- URLInput: HTML5 input type="url", validates format in browser
- TextInput: HTML5 input type="text", no format validation
- For YouTube URLs: TextInput allows youtu.be format

Migration Naming:
- Django auto-generated: 0004_alter_post_id_alter_post_youtube_url_and_more.py
- Also altered other fields for AutoField compatibility
- Safe to apply, only schema changes

Video ID Extraction:
- Method handles: youtu.be/, youtube.com/watch?v=, youtube.com/embed/
- Strips query parameters: ?si=, ?t=, &t=
- Validates: 9-12 character alphanumeric + dash/underscore
- Returns: None if invalid
